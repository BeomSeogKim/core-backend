# 스토리지 엔진 (Storage Engine)
MySQL은 다른 RDBMS(Oracle, PostgreSQL 등)와 달리 스토리지 엔진을 골라 사용할 수 있다.

대표적으로 InnoDB, MyISAM, Memory 엔진 등이 존재한다.

## InnoDB
MySQL 버전 5.5 이후부터 Default 엔진으로 채택 됨

핵심 특징
- 트랜잭션 지원
- 레코드 단위 잠금
    -  데이터 한 줄(Row)만 잠금
    -  같은 테이블에 대해 트랜잭션이 서로 별도의 row들을 처리할 수 있어, 동시성 처리에 적합
- MVCC (Multi Version Concurrency Control)
    - **잠금 없는 일관된 읽기**를 제공한다.
    - 누군가 데이터를 `UPDATE`하고 있어도 (아직 커밋 전), 다른 사람은 `Lock`을 대기하지 않고, 수정 전의 데이터 (Undo Log)를 바로 읽어간다.
    - 즉, 읽기 작업이 쓰기 작업을 방해하지 않는다.
- 클러스터링 인덱스
- 외래키 지원


## MyISAM
MySQL 버전 5.5 이전까지의 Default 엔진

핵심 특징
- 테이블 단위 잠금
    - 한 트랜잭션이 테이블의 특정 row 수정 시, 테이블 전체가 잠긴다.
- 트랜잭션 미지원
    - 작업 도중 에러가 나도 롤백이 되지 않음

_최근의 경우 InnoDB에서 MyISAM의 장점들을 다 지니고 있어 별도 사용하지 않음_

## Memory
디스크가 아닌 메모리에 데이터를 저장

핵심 특징
- 압도적인 속도
- 휘발성
- 사용 용도 : 임시 테이블, 세션 정보, 중요하지 않은 캐시 데이터 저장 시사용
