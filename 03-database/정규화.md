# 정규화 (Normalization)

## 핵심 개념

**정규화**는 데이터 중복을 제거하고 **이상(Anomaly)**을 방지하기 위해 테이블을 분리하는 과정이다. 삽입/수정/삭제 시 데이터 불일치가 발생하는 것을 막는다.

## 동작 원리

### 이상 현상 (Anomaly)

정규화가 안 된 테이블에서 발생하는 문제:

| 이상 | 설명 |
|------|------|
| **삽입 이상** | 새 데이터를 추가할 때 불필요한 데이터도 함께 넣어야 함 |
| **갱신 이상** | 중복 데이터 중 일부만 수정되어 불일치 발생 |
| **삭제 이상** | 데이터 삭제 시 의도하지 않은 다른 정보도 함께 삭제됨 |

### 1NF — 원자값 (Atomic Value)

각 컬럼은 **하나의 값만** 가져야 한다. 반복 그룹이나 다중 값을 허용하지 않는다.

```
위반:
| 학생  | 취미        |
|------|------------|
| 홍길동 | 축구, 농구   |  ← 하나의 셀에 여러 값

준수:
| 학생  | 취미 |
|------|-----|
| 홍길동 | 축구 |
| 홍길동 | 농구 |
```

### 2NF — 부분 종속 제거

**복합 PK**일 때, PK의 **일부에만 종속**되는 컬럼을 별도 테이블로 분리한다. (1NF 만족 전제)

```
위반: PK(학생ID, 과목ID) → 학생이름
      학생이름은 학생ID만으로 결정됨 (부분 종속)

┌──────────────────────────────────────┐
│ 학생ID | 과목ID | 학생이름 | 성적      │
│  1     │  101  │ 홍길동   │  A       │
│  1     │  102  │ 홍길동   │  B       │  ← "홍길동" 중복
└──────────────────────────────────────┘

준수: 학생이름을 학생 테이블로 분리
┌─────────────────┐   ┌─────────────────────────┐
│ 학생ID | 학생이름  │   │ 학생ID | 과목ID | 성적    │
│  1     │ 홍길동   │   │  1     │  101  │  A     │
└─────────────────┘   │  1     │  102  │  B     │
                      └─────────────────────────┘
```

### 3NF — 이행 종속 제거

PK가 아닌 컬럼이 **다른 비키 컬럼을 결정**하는 관계(A→B→C)를 분리한다. (2NF 만족 전제)

```
위반: 학생ID → 학과ID → 학과명 (이행 종속)

┌──────────────────────────────────┐
│ 학생ID | 학과ID | 학과명           │
│  1     │  CS   │ 컴퓨터공학       │
│  2     │  CS   │ 컴퓨터공학       │  ← "컴퓨터공학" 중복
└──────────────────────────────────┘

준수: 학과명을 학과 테이블로 분리
┌───────────────────┐   ┌──────────────────┐
│ 학생ID | 학과ID     │   │ 학과ID | 학과명    │
│  1     │  CS       │   │  CS   │ 컴퓨터공학 │
│  2     │  CS       │   └──────────────────┘
└───────────────────┘
```

### BCNF — 모든 결정자가 후보키

3NF를 강화한 형태. **모든 결정자(다른 컬럼의 값을 결정하는 컬럼)가 후보키**여야 한다.

### 정규형 요약

| 정규형 | 제거 대상 | 핵심 |
|--------|----------|------|
| **1NF** | 반복/다중값 | 컬럼은 원자값만 |
| **2NF** | 부분 종속 | 복합 PK 일부에만 종속되는 컬럼 분리 |
| **3NF** | 이행 종속 | 비키 → 비키 종속 분리 |
| **BCNF** | 후보키가 아닌 결정자 | 모든 결정자가 후보키 |

### 반정규화 (Denormalization)

정규화를 역으로 되돌려 **의도적으로 중복을 허용**하는 것. 읽기 성능 향상이 필요할 때 JOIN을 줄이기 위해 사용한다. 데이터 일관성과 성능 사이의 트레이드오프.
