# B-Tree, B+Tree

## B-Tree vs B+Tree
### B-Tree (Balanced Tree)
하나의 노드가 가질 수 있는 자식 노드의 개수가 2개 이상인 트리 구조.

데이터가 추가되거나 삭제되어도 트리의 높이를 균형있게 유지하여 검색 성능을 $O(logN)$ 으로 보장한다.

구조적인 특징
- 데이터 저장 위치 : 모든 노드(루프, 내부, 리프)에 Key와 Data (혹은 포인터)를 저장한다.
- 탐색 : 찾고자 하는 Key 발견 시 리프 노드까지 내려가지 않고 바로 탐색을 종료할 수 있다.

단점
- 노드마다 데이터가 저장되므로 한 페이지에 저장할 수 있는 Key의 개수가 적다.
- 특정 범위의 데이터를 찾기 위해 트리의 여러 노드를 오르락내리락하며 탐색해야 하여 비효율적임


### B+Tree
B-Tree의 단점을 보완하여 DB 인덱싱에 최적화된 구조이다. (MySQL의 InnoDB)

구조적 특징
- 데이터 저장 위치 분리
    - 내부 노드 : 오로지 리프 노드로 가는 Key와 포인터 주소만 저장
    - 리프 노드 : 모든 Key와 실제 Data (혹은 포인터)가 여기에만 저장이 된다.
- 리프 노드의 연결
    - 모든 리프 노드는 Linked List (InnoDB : Doubly Linked List)로 서로 연결되어 있다.
