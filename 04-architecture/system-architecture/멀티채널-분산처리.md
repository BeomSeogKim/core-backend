# 멀티 채널 분산 처리 (Router 패턴)

## 핵심 개념

단일 큐에서 여러 채널(국가, 플랫폼 등)을 순차 처리하면 채널 수 증가 시 처리 시간이 선형으로 증가한다. **Router 패턴**은 항목별 연동 채널을 확인하고 채널별 전용 큐에 분산하여, 각 Consumer가 독립적으로 병렬 처리하는 구조이다.

```
기존:
goodsNo → [채널1 처리 → 채널2 처리 → 채널3 처리 ...] 순차
                        ↑
            채널 수 증가 = 처리 시간 선형 증가
            병목: 외부 API I/O 대기 누적
```

> [!note] OCP 준수
> 채널 추가 시 기존 코드 수정 없이 채널 전용 큐 설정과 Consumer 클래스 1개만 추가하면 된다. **Open-Closed Principle**을 자연스럽게 충족하는 구조이다.

## 동작 원리

### Router 패턴 구조

```
Publisher → Router (채널 확인)
                ↓ 분산
    채널1 큐  채널2 큐  채널3 큐  ...
        ↓         ↓        ↓
    각 Consumer 독립적 병렬 처리
```

- **Publisher**: 처리할 항목을 Router에 전달
- **Router**: 항목별 연동 채널을 확인하고 채널별 전용 큐에 분산
- **Consumer**: 채널별 독립 Consumer가 병렬로 처리

### Promise.all vs Promise.allSettled

| | `Promise.all()` | `Promise.allSettled()` |
|--|----------------|----------------------|
| 실패 시 | 하나라도 실패하면 전체 reject (fail-fast) | 모두 기다리고 성공/실패 각각 반환 |
| 사용 시점 | 하나라도 실패하면 전체 롤백이 필요할 때 | 부분 실패 허용, 각각의 결과가 필요할 때 |

> [!tip] Promise.all로 allSettled 동작 구현
> 내부 try-catch로 reject를 막으면 `Promise.all()`로도 `Promise.allSettled()` 동작을 구현할 수 있다. 아래 코드 예시 참조.

### 실패 처리 전략

```
처리 중 실패 발생
    ↓
SyncRecord에 기록 (goodsNo + 에러 정보)
    ↓
에러 타입 분류
    - 5xx (일시적 오류) → 배치 재시도 대상
    - 4xx (영구적 오류) → 재시도 없음
    ↓
배치 스케줄러 (N시간 주기) → 5xx 실패 건 재시도
```

> [!warning] 재시도 제한 필요
> 재시도 횟수 제한 없을 경우, 외부 서비스 장애 시 Rate Limit / 블랙리스트 위험이 있다. 최대 재시도 횟수 제한 + Dead Letter 처리를 적용해야 한다. [[2-Areas/backend/04-architecture/design-patterns/Exponential-Backoff-Circuit-Breaker|Exponential Backoff & Circuit Breaker]] 패턴과 조합하면 더 효과적이다.

### 확장성 설계

채널 추가 시:
1. 채널 전용 큐 설정 추가
2. 해당 채널 Consumer 클래스 1개 추가

기존 코드 수정 없이 확장 가능

## 코드 예시

### 병렬 처리 구현 (TypeScript)

```typescript
// Promise.all() + 내부 try-catch → 실질적 Promise.allSettled() 동작
await Promise.all(
    goodsNos.map(async (goodsNo) => {
        try {
            await externalApi.register(goodsNo);
            syncRecord.success(goodsNo);
        } catch (error) {
            syncRecord.fail(goodsNo, error); // 예외를 삼키고 기록만
        }
    })
);
```

**핵심**: 예외를 내부에서 catch해 reject를 전파하지 않음 → 하나의 실패가 전체에 영향 없음

### 실전 적용 예시 (eBay 글로벌 연동)

```
product-publisher → Router (연동 국가 확인)
                        ↓ 분산
              US 큐  AU 큐  DE 큐  FR 큐  IT 큐  GB 큐
                ↓      ↓      ↓      ↓      ↓      ↓
            각 국가 Consumer 독립적 병렬 처리
```

- 기존: 6개국 순차 처리 → 국가 수만큼 처리 시간 증가
- 개선: 국가별 독립 큐 → 국가 수 증가해도 처리 시간 유지
- 한 국가 실패 → 다른 국가 영향 없음
