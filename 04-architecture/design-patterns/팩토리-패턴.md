# 팩토리 패턴 (Factory Pattern)

## 핵심 개념

**생성 패턴(Creational Pattern)** 의 하나. **객체 생성 책임을 별도 클래스(팩토리)로 분리**해 호출부가 구체 클래스에 의존하지 않도록 한다.

- `new 구체클래스()` 직접 호출 대신 팩토리에 위임
- 생성 로직을 한 곳에 집중 → OCP(확장에 열림, 변경에 닫힘) 준수

## 동작 원리

### 문제 상황

```java
// 결제 수단 추가 시 모든 if-else 분기를 수정해야 함
if (type.equals("CARD")) {
    payment = new CardPayment();
} else if (type.equals("KAKAO")) {
    payment = new KakaoPayment();
} else if (type.equals("NAVER")) {  // 추가 시 코드 변경 필요
    payment = new NaverPayment();
}
```

이 분기가 여러 곳에 퍼지면 **산탄총 수술(Shotgun Surgery)** 문제 발생.

### Simple Factory

```java
public interface Payment {
    void pay(int amount);
}

public class PaymentFactory {
    public static Payment create(String type) {
        return switch (type) {
            case "CARD"  -> new CardPayment();
            case "KAKAO" -> new KakaoPayment();
            case "NAVER" -> new NaverPayment();
            default -> throw new IllegalArgumentException("Unknown type: " + type);
        };
    }
}

// 호출부 — 생성 방법 불필요
Payment payment = PaymentFactory.create("KAKAO");
payment.pay(10000);
```

### Factory Method Pattern

부모 클래스가 객체 생성 인터페이스를 정의하고, **서브클래스가 어떤 객체를 생성할지 결정**한다.

```java
// 추상 팩토리 메서드
public abstract class PaymentCreator {
    public void processPayment(int amount) {
        Payment payment = createPayment();  // 팩토리 메서드
        payment.validate();
        payment.pay(amount);
        payment.notify();
    }
    protected abstract Payment createPayment();  // 서브클래스가 구현
}

public class KakaoPaymentCreator extends PaymentCreator {
    protected Payment createPayment() {
        return new KakaoPayment();
    }
}
```

### 종류 비교

| 종류 | 특징 | 사용 시점 |
|------|------|---------|
| **Simple Factory** | 팩토리 클래스에 생성 로직 집중 | 단순한 생성 분기 |
| **Factory Method** | 서브클래스가 생성 결정 | 생성 로직을 서브클래스에 위임 |
| **Abstract Factory** | 연관 객체 군을 한꺼번에 생성 | 제품군 단위 생성 |

## 코드 예시

```java
// Spring BeanFactory — 팩토리 패턴의 대표 예
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);
UserService userService = context.getBean(UserService.class);  // 팩토리가 생성 담당

// 실무 활용 — 알림 발송 팩토리
public class NotificationFactory {
    public static Notification create(NotificationType type) {
        return switch (type) {
            case EMAIL -> new EmailNotification();
            case SMS   -> new SmsNotification();
            case PUSH  -> new PushNotification();
        };
    }
}
```
