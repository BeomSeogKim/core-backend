# 프록시 패턴 (Proxy Pattern)

## 핵심 개념

**구조 패턴(Structural Pattern)** 의 하나. **원본 객체를 감싸는 대리 객체(Proxy)** 를 두어 부가 기능을 추가하는 패턴.

- 프록시는 원본과 **같은 인터페이스** 구현 → 클라이언트는 프록시 여부를 모름
- 실제 작업은 원본 객체에 **위임(delegate)**
- 부가 기능: 로깅, 캐싱, 접근 제어, 지연 초기화, 트랜잭션 등

## 동작 원리

```
Client
  │
  ▼
[Proxy]                    [RealObject]
  - target: RealObject ──→  - 실제 비즈니스 로직
  - method()
      │ 사전 작업 (로깅, 인증 등)
      │ target.method()  ──→ 위임
      │ 사후 작업 (트랜잭션 커밋 등)
      ▼
```

### 구현 예시

```java
public interface UserService {
    User findById(Long id);
    void save(User user);
}

// 실제 객체
public class UserServiceImpl implements UserService {
    public User findById(Long id) {
        return userRepository.findById(id);
    }
    public void save(User user) {
        userRepository.save(user);
    }
}

// 프록시 — 로깅 추가
public class UserServiceProxy implements UserService {
    private final UserService target;
    
    public UserServiceProxy(UserService target) {
        this.target = target;
    }
    
    public User findById(Long id) {
        log.info("findById 호출: id={}", id);
        long start = System.currentTimeMillis();
        
        User user = target.findById(id);    // 실제 객체에 위임
        
        log.info("실행 시간: {}ms", System.currentTimeMillis() - start);
        return user;
    }
    
    public void save(User user) {
        target.save(user);  // 위임
    }
}
```

### JDK Dynamic Proxy vs CGLIB

Spring AOP는 런타임에 프록시를 자동 생성한다.

| | JDK Dynamic Proxy | CGLIB |
|---|---|---|
| 조건 | **인터페이스 필요** | 인터페이스 없어도 가능 |
| 방식 | 인터페이스 기반 | **클래스 상속** |
| Spring Boot 기본 | — | **CGLIB** (2.0+) |

```java
// JDK Dynamic Proxy — 런타임에 프록시 생성
UserService proxy = (UserService) Proxy.newProxyInstance(
    UserService.class.getClassLoader(),
    new Class[]{UserService.class},
    (proxyObj, method, args) -> {
        log.info("before: {}", method.getName());
        Object result = method.invoke(target, args);
        log.info("after: {}", method.getName());
        return result;
    }
);
```

### 프록시 패턴 vs 데코레이터 패턴

두 패턴은 구조가 동일하지만 **의도**가 다르다.

| | 프록시 | 데코레이터 |
|---|---|---|
| 의도 | **접근 제어** (원본 대리) | **기능 확장** (원본에 기능 추가) |
| 원본 참조 | 프록시가 직접 생성/보유 | 외부에서 주입 |
| 예시 | Spring AOP, 캐싱 프록시 | Java I/O Stream 래핑 |

## 코드 예시

```java
// Spring @Transactional = 프록시 패턴
@Service
public class OrderService {
    
    @Transactional
    public void placeOrder(Order order) {
        // Spring이 프록시로 감싸서 트랜잭션 시작/커밋/롤백 처리
        orderRepository.save(order);
        paymentService.charge(order.getAmount());
    }
}

// 주의: Self-invocation 시 프록시 미적용
@Service
public class UserService {
    public void outer() {
        inner();  // 프록시 안 거침 → @Transactional 미적용
    }
    
    @Transactional
    public void inner() { ... }
}
```
