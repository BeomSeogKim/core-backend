# 전략 패턴 (Strategy Pattern)

## 핵심 개념

**행동 패턴(Behavioral Pattern)** 의 하나. **알고리즘(행동)을 인터페이스로 캡슐화**해 런타임에 교체 가능하게 하는 패턴.

- 변하는 것(알고리즘)을 인터페이스로 분리
- 컨텍스트(Context)는 전략을 주입받아 사용 → 구체 구현을 모름
- OCP 준수: 새 전략 추가 시 컨텍스트 수정 없음

## 동작 원리

```
[Context]
  - strategy: Strategy  ←──── [Strategy Interface]
  - execute()                       ↑         ↑
       │                    [ConcreteA]  [ConcreteB]
       └→ strategy.execute()
```

### 구현 예시 — 결제 전략

```java
// 전략 인터페이스
public interface PaymentStrategy {
    void pay(int amount);
}

// 구체 전략들
public class CardStrategy implements PaymentStrategy {
    private final String cardNumber;
    
    public CardStrategy(String cardNumber) {
        this.cardNumber = cardNumber;
    }
    
    public void pay(int amount) {
        System.out.println("카드 결제: " + amount + "원 (" + cardNumber + ")");
    }
}

public class KakaoStrategy implements PaymentStrategy {
    public void pay(int amount) {
        System.out.println("카카오페이 결제: " + amount + "원");
    }
}

// 컨텍스트 — 전략을 주입받아 사용
public class PaymentService {
    private PaymentStrategy strategy;
    
    public PaymentService(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    // 런타임에 전략 교체 가능
    public void changeStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void checkout(int amount) {
        strategy.pay(amount);  // 어떤 전략인지 몰라도 됨
    }
}

// 사용
PaymentService service = new PaymentService(new CardStrategy("1234-5678"));
service.checkout(10000);

service.changeStrategy(new KakaoStrategy());  // 런타임 교체
service.checkout(5000);
```

### 전략 패턴 vs 팩토리 패턴

| | 전략 패턴 | 팩토리 패턴 |
|---|---|---|
| 관심사 | **행동(알고리즘)** 캡슐화 | **객체 생성** 캡슐화 |
| 목적 | 알고리즘 교체 | 생성 분기 집중 |
| 런타임 변경 | **가능** | 어려움 |

### 전략 패턴 vs 템플릿 메서드 패턴

| | 전략 | 템플릿 메서드 |
|---|---|---|
| 구조 | **구성(주입)** | **상속** |
| 흐름 제어 | 컨텍스트가 위임 | 부모가 고정 |
| 런타임 교체 | **가능** | 불가 (상속 기반) |

## 코드 예시

```java
// Spring DI = 전략 패턴
// 어떤 구현체를 주입하느냐에 따라 행동 결정
@Service
public class NotificationService {
    private final NotificationSender sender;  // 인터페이스에 의존
    
    public NotificationService(NotificationSender sender) {
        this.sender = sender;
    }
    
    public void send(String message) {
        sender.send(message);
    }
}

// 테스트: MockSender 주입
// 이메일: EmailSender 주입
// SMS: SmsSender 주입
```
