# 템플릿 메서드 패턴 (Template Method Pattern)

## 핵심 개념

**행동 패턴(Behavioral Pattern)** 의 하나. **알고리즘의 골격(순서)을 부모 클래스에서 고정**하고, 달라지는 부분만 서브클래스가 구현하도록 하는 패턴.

- 공통 흐름은 부모의 `final` 메서드로 고정 → 순서 변경 불가
- 변하는 부분은 `abstract` 메서드로 선언 → 서브클래스가 구현
- 코드 중복 제거 + 알고리즘 구조 일관성 유지

## 동작 원리

```
[AbstractClass]
  + templateMethod() — final, 흐름 고정
      │ step1()  — 공통, 구현됨
      │ step2()  — abstract, 서브클래스 구현
      │ step3()  — 공통, 구현됨
      │ step4()  — abstract, 서브클래스 구현

[ConcreteA]               [ConcreteB]
  + step2() { A 방식 }     + step2() { B 방식 }
  + step4() { A 방식 }     + step4() { B 방식 }
```

### 구현 예시

```java
// 추상 클래스 — 알고리즘 골격 정의
public abstract class Beverage {
    
    // 템플릿 메서드 — 흐름 고정 (final로 오버라이드 방지)
    public final void prepare() {
        boilWater();
        brew();
        pourInCup();
        if (customerWantsCondiments()) {  // Hook 메서드 (선택적 오버라이드)
            addCondiments();
        }
    }
    
    // 공통 구현
    private void boilWater() {
        System.out.println("물 끓이기");
    }
    
    private void pourInCup() {
        System.out.println("컵에 붓기");
    }
    
    // 추상 메서드 — 서브클래스 필수 구현
    protected abstract void brew();
    protected abstract void addCondiments();
    
    // Hook 메서드 — 서브클래스 선택적 오버라이드 (기본값 제공)
    protected boolean customerWantsCondiments() {
        return true;
    }
}

// 서브클래스 — 달라지는 부분만 구현
public class Coffee extends Beverage {
    protected void brew() {
        System.out.println("필터로 커피 내리기");
    }
    protected void addCondiments() {
        System.out.println("설탕과 우유 추가");
    }
}

public class Tea extends Beverage {
    protected void brew() {
        System.out.println("찻잎 우리기");
    }
    protected void addCondiments() {
        System.out.println("레몬 추가");
    }
    
    @Override
    protected boolean customerWantsCondiments() {
        return false;  // 차는 첨가물 없이
    }
}
```

### Hook 메서드

기본 구현을 제공하지만 서브클래스가 선택적으로 오버라이드할 수 있는 메서드. 알고리즘의 특정 지점에서 서브클래스가 끼어들 수 있는 확장 포인트다.

### 전략 패턴과의 비교

| | 템플릿 메서드 | 전략 |
|---|---|---|
| 구조 | **상속** | **구성(주입)** |
| 알고리즘 변경 | 컴파일 타임 (상속으로 결정) | **런타임 교체** 가능 |
| 흐름 제어 | **부모가 고정** | 컨텍스트가 위임 |
| 결합도 | 높음 (상속 관계) | 낮음 (인터페이스 의존) |

## 코드 예시

```java
// Spring JdbcTemplate — 템플릿 메서드 패턴
// 공통: 연결, 예외처리, 자원반납
// 가변: SQL 쿼리 (개발자가 구현)
jdbcTemplate.query(
    "SELECT * FROM users WHERE id = ?",
    (rs, rowNum) -> new User(rs.getLong("id"), rs.getString("name")),
    id
);

// Spring의 AbstractController, AbstractRoutingDataSource 등
// 공통 흐름은 Abstract 클래스가 관리
// 달라지는 부분만 서브클래스가 구현
public class ReadWriteDataSource extends AbstractRoutingDataSource {
    @Override
    protected Object determineCurrentLookupKey() {
        return TransactionSynchronizationManager.isCurrentTransactionReadOnly()
            ? "read" : "write";
    }
}
```
