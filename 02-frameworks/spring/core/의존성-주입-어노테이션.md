# 의존성 주입 어노테이션 비교

## 핵심 개념

Spring에서 의존성 주입에 사용하는 어노테이션은 세 가지다. 출처와 주입 기준이 다르며, 실무에서는 생성자 주입 + Lombok 조합이 사실상 표준이다.

| 어노테이션 | 패키지 | 출처 | 주입 기준 |
|---|---|---|---|
| `@Autowired` | `org.springframework.beans.factory.annotation` | Spring 전용 | **타입** → 이름 fallback |
| `@Inject` | `jakarta.inject` | JSR-330 (Java 표준) | **타입** → 이름 fallback |
| `@Resource` | `jakarta.annotation` | JSR-250 (Java 표준) | **이름** → 타입 fallback |

## 동작 원리

### @Autowired

`AutowiredAnnotationBeanPostProcessor`가 처리한다. 타입으로 먼저 Bean을 찾고, 동일 타입이 여러 개면 필드명/파라미터명으로 매칭한다.

```java
@Service
public class OrderService {
    private final PaymentGateway paymentGateway;

    @Autowired  // 생성자가 하나면 생략 가능 (Spring 4.3+)
    public OrderService(PaymentGateway paymentGateway) {
        this.paymentGateway = paymentGateway;
    }
}
```

동일 타입 Bean이 여러 개일 때 우선순위: `@Qualifier` > `@Primary` > 필드명 매칭

### @Resource

`CommonAnnotationBeanPostProcessor`가 처리한다. 이름(name)으로 먼저 Bean을 찾고, 이름이 없으면 타입으로 fallback한다.

```java
@Resource(name = "kakaoPayGateway")  // Bean 이름 직접 지정
private PaymentGateway paymentGateway;
```

### @Inject

`AutowiredAnnotationBeanPostProcessor`가 처리한다. `@Autowired`와 동작이 거의 동일하지만 `required` 옵션이 없다.

```java
@Inject  // @Autowired(required = false) 같은 옵션 없음
private PaymentGateway paymentGateway;
```

### @Autowired vs @Inject 차이

| | `@Autowired` | `@Inject` |
|---|---|---|
| 출처 | Spring 전용 | Java 표준 (JSR-330) |
| `required` 옵션 | `required = false` 지원 | 없음 |
| 프레임워크 독립 | ❌ | ✅ |

## 실무 선택

실무에서는 세 어노테이션 모두를 거의 쓰지 않는다. 생성자 주입 + Lombok을 사용하면 어노테이션 없이도 의존성 주입이 동작한다.

```java
@Service
@RequiredArgsConstructor        // final 필드로 생성자 자동 생성
public class OrderService {
    private final OrderRepository orderRepository;   // @Autowired 불필요
    private final PaymentGateway paymentGateway;
}
```

`@Resource` / `@Inject`를 굳이 쓸 이유:
- Spring 컨테이너를 쓰면서 Java 표준을 선택할 이점이 없음
- Lombok과의 조합이 없어서 불편
