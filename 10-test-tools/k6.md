# k6

## 핵심 개념

**Grafana Labs**에서 개발한 오픈소스 부하 테스트 도구. [공식 문서](https://k6.io/)

Go 언어로 작성되어 가볍고, JavaScript 기반 스크립팅으로 테스트 시나리오를 작성하며, **CI/CD 파이프라인**에 쉽게 통합할 수 있는 것이 주요 특징이다.

> [!note]
> k6는 Go 런타임 위에서 JavaScript를 실행하므로, 적은 리소스로 많은 **Virtual Users**를 생성하여 높은 부하를 발생시킬 수 있다.

## 동작 원리

### 실행 단계

k6 스크립트는 크게 4가지 단계로 실행된다.

```
1. Init      → 패키지 import, 옵션 설정
      ↓
2. Setup     → 테스트 시작 전 한 번만 실행 (e.g. 토큰 발급)
      ↓
3. VU Code   → 실제 부하를 주는 로직 (default function, 반복 실행)
      ↓
4. Teardown  → 테스트 종료 후 한 번만 실행
```

### 핵심 특징

| 특징 | 설명 |
|------|------|
| JavaScript 기반 | JMeter처럼 XML/GUI 없이 코드로 테스트 작성 및 버전 관리 |
| 가벼운 성능 | Go 언어 기반, 적은 리소스로 높은 부하 생성 |
| CLI 기반 | 터미널에서 실행, CI/CD 자동화에 적합 |
| 다양한 결과 출력 | Stdout, JSON, Grafana, CloudWatch 등 실시간 전송 가능 |

> [!tip]
> `thresholds` 옵션으로 SLA(Service Level Agreement)를 코드로 정의할 수 있다. 예: `http_req_duration: ['p(95)<500']` - 95%의 요청이 500ms 이내여야 테스트 통과.

### JMeter vs k6

| 특성 | JMeter | k6 |
|------|--------|-----|
| 기반 언어 | Java | Go (스크립트: JS) |
| 인터페이스 | GUI | CLI / Code |
| 성능 | 리소스 효율성 낮음 | 리소스 효율성 좋음 |
| 보고서 | HTML 리포트 | Stdout, JSON, Grafana 등 실시간 전송 |

## 코드 예시

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

// 1. 설정 (Options)
export const options = {
  vus: 10,           // 가상 유저 10명
  duration: '30s',   // 30초 동안 테스트 진행
  thresholds: {      // 성공/실패 기준 (SLA 설정)
    http_req_duration: ['p(95)<500'], // 95%의 요청이 500ms 이내여야 함
  },
};

// 2. VU 로직 (Main function)
export default function () {
  const res = http.get('http://test.k6.io');

  // 응답 확인 (Assertion)
  check(res, {
    'status is 200': (r) => r.status === 200,
  });

  sleep(1); // 1초 대기 (User Think Time 시뮬레이션)
}
```

> [!warning]
> k6는 브라우저 환경이 아닌 Go 런타임에서 JavaScript를 실행하므로, `window`, `document` 등 브라우저 API는 사용할 수 없다. Node.js 모듈도 직접 import할 수 없다.
