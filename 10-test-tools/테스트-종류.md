# 테스트의 종류

## 핵심 개념

소프트웨어 테스트는 검증 범위와 목적에 따라 **단위 테스트, 통합 테스트, E2E 테스트, 성능 테스트** 등으로 나뉜다. 일반적으로 **테스트 피라미드** 원칙에 따라 단위 테스트를 가장 많이, E2E 테스트를 가장 적게 작성한다.

### Given-When-Then 패턴

테스트 코드를 구조화하는 표준 패턴:

```java
@Test
void 주문_생성_성공() {
    // given — 준비 (테스트에 필요한 데이터/상태 설정)
    Product product = new Product("상품A", 1000);

    // when — 실행 (테스트 대상 행동 수행)
    Order order = orderService.createOrder(product, 3);

    // then — 검증 (결과가 기대한 대로인지 확인)
    assertEquals(3000, order.getTotalPrice());
}
```
