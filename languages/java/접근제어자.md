# 접근제어자 (Access Modifier)

## 핵심 개념

**Access Modifier(접근제어자)**는 [[Class]]의 멤버(필드, 메서드, 생성자)와 클래스 자체에 대한 접근 범위를 제어하는 키워드다. **캡슐화(Encapsulation)**를 실현하는 핵심 도구이며, `private`, `default`, `protected`, `public` 4가지가 있다.

> [!note] 접근 범위 순서
> `private` < `default(package-private)` < `protected` < `public` 순으로 접근 범위가 넓어진다.

## 동작 원리

### private

- 같은 클래스 내에서만 접근 가능
- 가장 제한적인 접근 수준
- 캡슐화의 핵심 (외부에서 내부 구현 숨김)

### default (package-private)

- 접근 제어자를 명시하지 않은 경우
- 같은 패키지 내에서만 접근 가능
- 패키지 외부에서는 접근 불가

### protected

- 같은 패키지 + 다른 패키지의 자식 클래스에서 접근 가능
- 상속 관계를 고려한 접근 수준

### public

- 어디서든 접근 가능
- 가장 개방적인 접근 수준
- API로 공개할 부분에만 사용

| 접근제어자 | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
|-----------|:---------:|:---------:|:---------:|:---:|
| `private` | O | X | X | X |
| `default` | O | O | X | X |
| `protected` | O | O | O | X |
| `public` | O | O | O | O |

> [!warning] default와 protected의 혼동
> `default`는 같은 패키지 내에서만 접근 가능하고, `protected`는 같은 패키지 **+ 다른 패키지의 자식 클래스**까지 접근 가능하다. 상속이 없다면 둘의 동작은 동일하다.

## 코드 예시

```java
public class Account {
    private long balance;           // 같은 클래스만
    int accountNumber;              // 같은 패키지 (default)
    protected String ownerName;     // 같은 패키지 + 자식 클래스
    public String bankName;         // 어디서든

    // private 필드는 getter/setter로 접근
    public long getBalance() {
        return balance;
    }

    public void deposit(long amount) {
        if (amount > 0) {
            this.balance += amount;
        }
    }
}
```

> [!tip] 최소 권한 원칙
> 클래스 설계 시 모든 필드를 `private`으로 시작하고, 필요한 만큼만 접근 범위를 넓히는 것이 좋다. 이는 [[값 복사]]에서의 방어적 복사와 함께 객체의 내부 상태를 안전하게 보호하는 전략이다.
