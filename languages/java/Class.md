# Class

## 핵심 개념

**Class**는 객체를 만들기 위한 설계도이자 템플릿이다. 속성(필드)과 행위(메서드)를 정의하며, 코드 상에 존재하는 개념적 존재다. 메모리에 딱 한 번만 로드되어 [[JVM]]의 Method Area(Metaspace)에 저장된다.

> [!note] 클래스 vs 객체 vs 인스턴스
> - **클래스**: 설계도 (코드에 존재)
> - **객체**: 클래스를 기반으로 실체화된 것을 지칭하는 포괄적 용어
> - **인스턴스**: `new` 키워드로 힙 메모리에 할당된 구체적 실체. 특정 클래스와의 소속 관계를 강조할 때 사용

## 동작 원리

### 클래스, 객체, 인스턴스의 관계

**클래스**
- 객체를 만들기 위한 설계도, 틀, 템플릿
- 속성(필드)과 행위(메서드)를 정의
- 코드 상에 존재하는 개념적 존재
- 메모리에 딱 한 번만 로드 (Method Area / Metaspace)

**객체**
- 클래스를 기반으로 실체화된 것 전체를 지칭하는 포괄적 용어
- 소프트웨어 세계에 구현할 대상
- 클래스의 타입으로 선언된 것
- 실체 + 개념 모두 포함하는 가장 넓은 의미

**인스턴스**
- 클래스를 통해 실제 메모리에 생성된 실체
- `new` 키워드로 힙 메모리에 할당된 구체적 존재
- _특정 클래스의 객체라는 관계를 강조할 때 사용_
- 'Car 클래스의 인스턴스'처럼 소속 관계 표현

> [!tip] 값의 전달 방식
> 클래스로 생성된 인스턴스는 Reference Type이므로, 메서드에 전달할 때 참조값이 복사된다. 자세한 내용은 [[값 복사]]를 참고한다. 또한 [[접근제어자]]를 통해 클래스 멤버의 접근 범위를 제어할 수 있다.

## 코드 예시

```java
public class Car {
    String model;
    int speed;

    void accelerate() { }
}
```

```java
Car myCar = new Car();
// myCar는 참조 변수
// new Car()로 생성된 것이 인스턴스
// 이 실체를 객체라고도 부름
```

> [!warning] 참조 변수와 인스턴스의 구분
> `myCar`는 참조 변수이지 객체 자체가 아니다. `new Car()`가 힙에 생성한 인스턴스의 주소를 저장하는 스택 변수일 뿐이다.
