# 값 복사

## 핵심 개념

Java는 항상 **값에 의한 전달(Pass by Value)**만 사용한다. **Primitive Type**은 실제 값 자체가 복사되고, **Reference Type**은 객체의 주소값(참조)이 복사된다.

> [!note] Java는 Pass by Reference가 없다
> C++과 달리 Java에는 참조에 의한 전달이 존재하지 않는다. Reference Type을 전달할 때도 참조 "값"이 복사되는 것이다. 따라서 메서드 내에서 참조 변수를 재할당해도 원본 참조에는 영향이 없다.

## 동작 원리

### Primitive Type 복사

- int, long, double, boolean 등
- 값 자체가 새 메모리 공간에 복사
- 메서드 내 변경이 원본에 영향 없음

### Reference Type 복사

- 객체를 가리키는 주소값이 복사되는 것
- 참조를 통한 객체 내부 상태 변경 -> 원본도 변경됨
- 참조 변수 자체를 재할당 -> 원본 참조는 영향 없음

### String의 특수성

- Reference Type이지만 **불변객체(Immutable)**
- 모든 연산이 새 String 객체 생성
- 원본 변경 불가능 -> Primitive처럼 동작

> [!warning] 얕은 복사의 함정
> `clone()`이나 단순 복사 생성자를 사용하면 얕은 복사가 수행된다. 중첩된 컬렉션이나 객체를 수정하면 원본도 함께 변경되므로, 완전한 독립성이 필요하면 깊은 복사를 구현해야 한다.

### 얕은 복사 vs 깊은 복사

**얕은 복사 (Shallow Copy)**
- 최상위 레벨 필드만 복사
- 중첩 객체는 같은 참조 공유
- `clone()` 기본 구현, 단순 복사 생성자
- 주의: _중첩 컬렉션 수정시 원본도 변경됨_

**깊은 복사 (Deep Copy)**
- 모든 레벨을 재귀적으로 복사
- 완전히 독립적인 객체 그래프 생성
- 구현 방법
  - 재귀적 복사 생성자 (타입 안정성 O, but 코드 길어짐)
  - 직렬화 이용 (간단, 성능 낮음, Serializable 필요)
  - Apache Commons SerializationUtils
  - JSON 라이브러리 활용

## 코드 예시

```java
// Primitive Type - 값 복사
int a = 10;
int b = a;  // 값 자체가 복사
b = 20;     // a는 여전히 10

// Reference Type - 참조값 복사
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;      // 주소값이 복사
arr2[0] = 99;           // arr1[0]도 99로 변경됨

// 참조 변수 재할당 - 원본 무관
int[] arr3 = arr1;
arr3 = new int[]{10, 20, 30};  // arr1에는 영향 없음
```

> [!tip] 방어적 복사
> [[Class]]의 필드를 외부에 노출할 때는 방어적 복사(Defensive Copy)를 통해 내부 상태가 의도치 않게 변경되는 것을 방지할 수 있다. 특히 컬렉션 필드는 `Collections.unmodifiableList()`를 활용하는 것도 방법이다.
