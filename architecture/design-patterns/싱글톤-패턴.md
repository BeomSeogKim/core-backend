# 싱글톤 패턴 (Singleton Pattern)

## 핵심 개념

**생성 패턴(Creational Pattern)** 의 하나. 클래스의 인스턴스를 **애플리케이션 전체에서 하나만** 생성하고, 전역 접근점을 제공한다.

- 생성자를 `private`으로 선언해 외부 인스턴스화를 차단
- `static` 메서드로 동일한 인스턴스 반환

## 동작 원리

### 구현 방식 비교

**1. Eager Initialization — 가장 단순**
```java
public class Singleton {
    private static final Singleton INSTANCE = new Singleton();
    private Singleton() {}
    public static Singleton getInstance() { return INSTANCE; }
}
```
- 클래스 로딩 시점에 즉시 생성. Thread-safe.
- 사용 여부와 무관하게 메모리 점유.

**2. Lazy Initialization — 멀티스레드 위험**
```java
public class Singleton {
    private static Singleton instance;
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {           // 두 스레드가 동시에 null 체크 가능
            instance = new Singleton();   // 인스턴스 2개 생성 위험
        }
        return instance;
    }
}
```
- 멀티스레드 환경에서 Race Condition 발생 가능.

**3. Double-Checked Locking**
```java
public class Singleton {
    private static volatile Singleton instance;
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {      // 락 진입 후 재확인
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```
- `volatile` 필수: CPU 캐시가 아닌 메인 메모리에서 직접 읽기 (가시성 보장)
- 인스턴스 생성 후엔 `synchronized` 블록 진입 안 함 → 성능 개선

**4. LazyHolder — 실무 권장**
```java
public class Singleton {
    private Singleton() {}
    
    private static class LazyHolder {
        private static final Singleton INSTANCE = new Singleton();
    }
    
    public static Singleton getInstance() {
        return LazyHolder.INSTANCE;
    }
}
```
- `LazyHolder` 클래스는 `getInstance()` 최초 호출 시 로딩됨 → 지연 초기화
- JVM의 클래스 로딩은 thread-safe 보장 → `synchronized` 불필요
- 성능 + 안전성 + 지연 초기화 모두 만족

### 멀티스레드 문제가 발생하는 이유

```
Thread A                      Thread B
if (instance == null)  →  →  if (instance == null)
                              instance = new Singleton()
instance = new Singleton()  ← 이미 생성됐지만 A도 생성
→ 인스턴스 2개 존재
```

## 코드 예시

```java
// Spring Bean은 기본적으로 Singleton scope
@Service
public class UserService {
    // Spring이 컨테이너당 하나의 인스턴스만 생성
    // 개발자가 Singleton 구현 불필요
}

// 설정 정보처럼 전역 공유 객체가 필요한 경우
public class AppConfig {
    private static class Holder {
        private static final AppConfig INSTANCE = new AppConfig();
    }
    public static AppConfig getInstance() { return Holder.INSTANCE; }
}
```
