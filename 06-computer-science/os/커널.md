# 커널 (Kernel)

## 핵심 개념

**커널(Kernel)**은 OS의 핵심 부분으로, 하드웨어 자원(CPU, 메모리, 디스크, 네트워크)을 관리하고 보호하는 역할을 한다. 일반 프로그램이 하드웨어에 직접 접근하지 못하게 막고, 반드시 커널을 통해서만 접근하도록 강제한다.

## 동작 원리

CPU는 두 가지 실행 모드를 하드웨어 수준에서 구분한다:

### System Call (시스템 콜)

프로그램(user mode)이 커널의 기능을 요청하는 인터페이스. 파일 읽기, 네트워크 요청, 프로세스 생성 등 하드웨어 접근이 필요한 모든 작업은 시스템 콜을 통해야 한다.

OS마다 시스템 콜이 다르다:
- **Linux**: `read()`, `write()`, `open()`, `fork()` 등 (POSIX 기반)
- **Windows**: `ReadFile()`, `WriteFile()`, `CreateProcess()` 등 (Win32 API)

시스템 콜 한 번에 user → kernel → user 왕복이 발생하고, 이 모드 전환 자체가 ~1-10μs의 오버헤드를 만든다.

### 인터럽트 (Interrupt)

프로그램의 의지와 상관없이, 외부 이벤트가 발생하면 CPU가 현재 작업을 중단하고 커널 모드로 전환하여 긴급 처리하는 메커니즘.

```
프로그램 실행 중 (User Mode)
    │
    │  ← 인터럽트 발생!
    │
    ▼
CPU가 현재 작업 중단
    │
    ▼
User Mode → Kernel Mode 전환
    │
    ▼
인터럽트 핸들러 (ISR) 실행
    │
    ▼
Kernel Mode → User Mode 복귀
    │
    ▼
원래 작업 재개
```

#### 인터럽트 종류

| 종류 | 발생 원인 | 예시 |
|------|-----------|------|
| **하드웨어 인터럽트** | 외부 장치가 CPU에 신호 | 키보드 입력, 마우스 클릭, 디스크 I/O 완료, 타이머 |
| **소프트웨어 인터럽트 (Trap)** | 프로그램 내부에서 발생 | 시스템 콜, 0으로 나누기, 잘못된 메모리 접근 |

#### 타이머 인터럽트와 컨텍스트 스위칭

OS는 하드웨어 타이머를 설정해두고, Time Quantum이 만료되면 타이머 인터럽트가 발생한다. 이것이 [[컨텍스트-스위칭]]을 트리거하는 핵심 메커니즘이다:

```
Thread-A 실행 중 (User Mode)
    │
    │  ← 타이머 인터럽트 발생 (Time Quantum 만료)
    │
    ▼
Kernel Mode 진입
    │
    ▼
OS 스케줄러: 다음 실행할 스레드 선택
    │
    ▼
컨텍스트 스위칭 (A 저장 → B 복원)
    │
    ▼
Thread-B 실행 (User Mode)
```
