# 캐시의 지역성 원리

## 핵심 개념

**지역성 원리(Principle of Locality)**는 프로그램이 메모리에 접근할 때 특정 패턴을 보인다는 관찰이다. **시간적 지역성**과 **공간적 지역성** 두 가지가 있으며, CPU 캐시가 효과적인 이론적 근거가 된다.

상위로 갈수록 빠르지만 용량이 작다. 캐시는 자주 쓰는 데이터를 상위 계층에 두어 평균 접근 시간을 줄인다. 이것이 효과적인 이유가 바로 지역성 원리다.

### 시간적 지역성 (Temporal Locality)

한 번 접근한 데이터는 **가까운 시간 내에 다시 접근**할 가능성이 높다.

```java
// 반복문의 변수 i, sum → 매 반복마다 재접근
for (int i = 0; i < 1000; i++) {
    sum += arr[i];  // sum, i는 시간적 지역성
}
```

- 반복문 카운터, 스택 변수, 자주 호출되는 함수
- LRU 캐시 교체 알고리즘이 이 원리를 활용

### 공간적 지역성 (Spatial Locality)

접근한 데이터의 **인접 주소**도 곧 접근할 가능성이 높다.

```java
// 배열 순회 → arr[0] 접근 후 arr[1], arr[2]... 순차 접근
int[] arr = new int[1000];
for (int i = 0; i < 1000; i++) {
    arr[i] = i * 2;  // 연속된 메모리 주소 접근
}
```

- 배열 순회, 순차 코드 실행, 구조체/객체 필드 접근
- CPU가 데이터를 가져올 때 **캐시 라인(보통 64B)** 단위로 주변 데이터까지 함께 로드하는 이유

### 캐시 라인과 공간적 지역성

```
메모리 주소:  [0x100] [0x104] [0x108] ... [0x13C]
              ─────────── 하나의 캐시 라인 (64B) ───────────

arr[0]에 접근 → 캐시 라인 전체(arr[0]~arr[15])가 캐시에 로드
→ arr[1]~arr[15] 접근 시 캐시 히트 (Cache Hit)
```
