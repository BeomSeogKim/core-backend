# 메모리 관리 전략

## 핵심 개념

OS가 프로세스에 메모리를 할당하는 방식으로 **페이징(Paging)**과 **세그멘테이션(Segmentation)**이 있다. 페이징은 고정 크기, 세그멘테이션은 논리적 가변 크기로 메모리를 분할한다. 현대 OS는 대부분 페이징 기반이다.

## 동작 원리

### 페이징 (Paging)

메모리를 **고정 크기(보통 4KB)** 단위로 분할한다.

- **페이지 (Page)**: 가상 메모리의 고정 크기 블록
- **프레임 (Frame)**: 물리 메모리의 고정 크기 블록
- 페이지 테이블이 페이지 → 프레임 매핑을 관리

```
가상 메모리 (페이지)          물리 메모리 (프레임)
┌────────┐  4KB          ┌────────┐
│ Page 0 │──────────────→│Frame 3 │
├────────┤               ├────────┤
│ Page 1 │──────────┐    │Frame 1 │←─┐
├────────┤          │    ├────────┤  │
│ Page 2 │───┐      │    │Frame 5 │  │
├────────┤   │      │    ├────────┤  │
│ Page 3 │   │      └───→│Frame 2 │  │
└────────┘   │           ├────────┤  │
             └──────────→│Frame 7 │  │
                         └────────┘  │
                                     │
             (연속일 필요 없음) ───────┘
```

- **장점**: 외부 단편화 없음. 고정 크기라 관리 단순
- **단점**: 내부 단편화 발생 가능 (4KB 블록에 3KB만 사용하면 1KB 낭비)

### 세그멘테이션 (Segmentation)

메모리를 **논리적 단위(세그먼트)**로 분할한다. 코드, 데이터, 힙, 스택 등 의미 있는 단위별로 크기가 다르다.

```
프로세스 메모리 세그먼트
┌───────────────────┐
│ Code Segment  2KB │
├───────────────────┤
│ Data Segment  1KB │
├───────────────────┤
│ Heap Segment  8KB │
├───────────────────┤
│ Stack Segment 4KB │
└───────────────────┘
(각 세그먼트 크기가 다름)
```

- **장점**: 논리적으로 의미 있는 단위. 세그먼트별 접근 권한 설정 가능
- **단점**: 외부 단편화 발생 (가변 크기 할당/해제 반복 → 빈 공간 조각남)

### 현대 OS의 선택

현대 OS(Linux, Windows)는 **페이징 기반**을 사용한다. 외부 단편화가 발생하지 않고, [[가상-메모리]]의 Demand Paging과 결합하여 효율적인 메모리 관리가 가능하기 때문이다.
