# 동기 · 비동기 · 블로킹 · 논블로킹

## 핵심 개념

**동기/비동기**와 **블로킹/논블로킹**은 서로 다른 축의 개념이다. 동기/비동기는 **작업 완료를 누가 신경 쓰는가**(결과 확인 주체), 블로킹/논블로킹은 **호출 시 제어권을 바로 돌려받는가**(호출자의 상태)를 구분한다.

## 동작 원리

### 동기 vs 비동기 — 결과 확인의 주체

- **동기(Synchronous)**: 호출자가 직접 작업 완료 여부를 확인한다. 결과가 돌아올 때까지 관심을 유지한다.
- **비동기(Asynchronous)**: 호출된 쪽이 작업 완료 시 콜백, 이벤트 등으로 알려준다. 호출자는 결과에 신경 쓰지 않고 넘어간다.

### 블로킹 vs 논블로킹 — 제어권 반환 여부

- **블로킹(Blocking)**: 호출된 함수가 작업을 완료할 때까지 제어권을 돌려주지 않는다. 호출한 스레드는 대기 상태.
- **논블로킹(Non-blocking)**: 호출된 함수가 즉시 제어권을 돌려준다. 호출한 스레드는 다른 작업 가능.

#### 동기 + 블로킹

가장 일반적인 방식. `read()` 시스템 콜 호출 → 데이터가 올 때까지 스레드가 멈춰서 대기.

```java
// Java의 전통적인 I/O
InputStream in = socket.getInputStream();
int data = in.read();  // 데이터 올 때까지 블로킹
```

#### 동기 + 논블로킹

호출 후 즉시 리턴받지만, 결과를 내가 직접 반복 확인 (**폴링, Polling**).

```java
// 반복적으로 완료 여부 확인
while (!future.isDone()) {
    // 다른 작업 가능
}
Result result = future.get();
```

#### 비동기 + 논블로킹

호출 후 즉시 리턴. 완료 시 콜백으로 통보. 가장 효율적.

```java
// Java NIO / CompletableFuture
CompletableFuture.supplyAsync(() -> fetchData())
    .thenAccept(result -> process(result));  // 콜백
// 호출 스레드는 바로 다음 작업 진행
```

#### 비동기 + 블로킹

비동기인데 블로킹이면 비동기의 이점이 사라진다. 실무에서 의도적으로 사용하는 경우는 드물다.
