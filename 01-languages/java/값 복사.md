# 값 복사

## 기본원칙
- 자바는 항상 **값에 의한 전달**만 사용
- Primitive Type: 실제 값 자체가 복사됨
- Reference Type: 객체의 주소값(참조)이 복사됨

**Primitive Type 복사**
- int, long, double, boolean 등..
- 값 자체가 새 메모리 공간에 복사
- 메서드 내 변경이 원본에 영향 없음

**Reference Type 복사**
- 객체를 가리키는 주소값이 복사되는 것
- 참조를 통한 객체 내부 상태 변경 -> 원본도 변경됨
- 참조 변수 자체를 재할당 -> 원본 참조는 영향 없음

**String의 특수성**
- Reference Type이지만 불변객체
- 모든 연산이 새 String 객체 생성
- 원본 변경 불가능 -> Primitive처럼 동작

## 얕은 복사 vs 깊은 복사

**얕은 복사 (Shallow Copy)**
- 최상위 레벨 필드만 복사
- 중첩 객체는 같은 참조 공유
- `clone()` 기본 구현, 단순 복사 생성자
- 주의: _중첩 컬렉션 수정시 원본도 변경됨_

**깊은 복사 (Deep copy)**
- 모든 레벨을 재귀적으로 복사
- 완전히 독립적인 객체 그래프 생성
- 구현 방법
  - 재귀적 복사 생성자 (타입 안정성 O, but 코드 길어짐)
  - 직렬화 이용 (간단, 성능 낮음, Serializable필요)
  - Apache Commons SerializationUtils
  - JSON 라이브러리 활용
