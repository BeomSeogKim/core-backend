# 제네릭 (Generics)

## 핵심 개념

**제네릭(Generics)**은 클래스, 인터페이스, 메서드를 정의할 때 **타입을 파라미터로** 지정하는 기능이다. 컴파일 타임에 타입 안전성을 보장하고, 불필요한 타입 캐스팅을 제거한다.

## 동작 원리

### 제네릭이 없으면

```java
// Object로 받아야 함 → 아무 타입이나 들어감
List list = new ArrayList();
list.add("hello");
list.add(123);                    // 의도하지 않은 타입도 들어감

String s = (String) list.get(0);  // 매번 캐스팅 필요
String s2 = (String) list.get(1); // 런타임에 ClassCastException!
```

문제점:
- **타입 안전성 없음** — 잘못된 타입이 들어가도 컴파일 시 발견 불가
- **캐스팅 필요** — 꺼낼 때마다 타입 캐스팅 필요
- **런타임 에러** — `ClassCastException`이 런타임에 발생

### 제네릭 사용

```java
// 컴파일 타임에 타입 제한
List<String> list = new ArrayList<>();
list.add("hello");
list.add(123);           // 컴파일 에러! Integer는 넣을 수 없음

String s = list.get(0);  // 캐스팅 불필요
```

### 제네릭 클래스와 메서드

```java
// 제네릭 클래스
public class Box<T> {
    private T item;

    public void set(T item) { this.item = item; }
    public T get() { return item; }
}

Box<String> stringBox = new Box<>();
stringBox.set("hello");
String value = stringBox.get();  // 캐스팅 불필요

Box<Integer> intBox = new Box<>();
intBox.set(42);
```

```java
// 제네릭 메서드
public <T> T getFirst(List<T> list) {
    return list.get(0);
}

String first = getFirst(List.of("a", "b"));   // T → String 추론
Integer num = getFirst(List.of(1, 2, 3));      // T → Integer 추론
```

### 바운디드 타입 (Bounded Type)

제네릭 타입에 제한을 걸 수 있다.

```java
// T는 Number 또는 그 하위 클래스만 허용
public <T extends Number> double sum(List<T> list) {
    double total = 0;
    for (T item : list) {
        total += item.doubleValue();  // Number의 메서드 사용 가능
    }
    return total;
}

sum(List.of(1, 2, 3));       // OK — Integer extends Number
sum(List.of(1.5, 2.5));      // OK — Double extends Number
sum(List.of("a", "b"));      // 컴파일 에러! String은 Number가 아님
```

### 와일드카드 (Wildcard)

```java
// 읽기 전용 — 어떤 타입이든 받을 수 있지만 추가 불가
public void printAll(List<? extends Number> list) {
    for (Number n : list) {
        System.out.println(n);  // 읽기 OK
    }
    // list.add(1);  // 컴파일 에러! 정확한 타입을 모르므로 추가 불가
}

// 쓰기 가능 — Number 또는 그 상위 타입의 리스트
public void addNumbers(List<? super Integer> list) {
    list.add(1);    // 쓰기 OK
    list.add(2);
    // Integer n = list.get(0);  // 읽기 시 Object로만 받을 수 있음
}
```

**PECS 원칙**: **P**roducer **E**xtends, **C**onsumer **S**uper
- 데이터를 **읽을 때** (Producer) → `<? extends T>`
- 데이터를 **넣을 때** (Consumer) → `<? super T>`

### 타입 소거 (Type Erasure)

제네릭은 **컴파일 타임에만 존재**하고, **런타임에는 제거**된다. 컴파일러가 타입 체크를 수행한 후, 바이트코드에서는 제네릭 정보를 지우고 Object(또는 바운드 타입)로 대체한다.

```java
// 컴파일 전
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);

// 컴파일 후 (타입 소거)
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 컴파일러가 캐스팅 삽입
```

**타입 소거로 인한 제약:**
- `new T()` — 불가 (런타임에 T가 뭔지 모름)
- `instanceof T` — 불가 (런타임에 T 정보 없음)
- `new T[]` — 불가 (제네릭 배열 생성 불가)

타입 소거가 존재하는 이유는 **하위 호환성** 때문이다. 제네릭이 없던 Java 5 이전 코드와의 호환을 유지하기 위해 런타임에는 제네릭 정보를 제거한다.
