# 직렬화 / 역직렬화

## 핵심 개념

**직렬화(Serialization)**는 메모리의 객체를 **바이트 스트림으로 변환**하여 저장·전송 가능한 형태로 만드는 것이다. **역직렬화(Deserialization)**는 바이트 스트림을 다시 객체로 복원하는 과정이다.

## 동작 원리

### 왜 필요한가?

메모리의 객체는 해당 JVM 프로세스 내에서만 존재한다. 이 객체를 네트워크로 전송하거나, 파일/DB에 저장하려면 **바이트 스트림으로 변환**해야 한다.

```
           직렬화                              역직렬화
객체 ──────────────→ 바이트 스트림 ──────────────→ 객체
(메모리)              (파일/네트워크/DB)            (메모리)
```

사용처:
- 네트워크 통신 (API 호출, RPC)
- 파일/DB 저장
- 캐시 저장 (Redis 등)
- 세션 클러스터링

### Java 기본 직렬화 — Serializable

`Serializable` 인터페이스를 구현하면 Java 기본 직렬화를 사용할 수 있다.

```java
public class User implements Serializable {
    private static final long serialVersionUID = 1L;  // 버전 관리

    private String name;
    private int age;
    private transient String password;  // transient → 직렬화에서 제외
}
```

```java
// 직렬화 — 객체 → 바이트 스트림
try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("user.dat"))) {
    oos.writeObject(new User("홍길동", 25));
}

// 역직렬화 — 바이트 스트림 → 객체
try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("user.dat"))) {
    User user = (User) ois.readObject();
}
```

#### transient 키워드

직렬화에서 **제외**할 필드에 사용한다. 역직렬화 시 해당 필드는 기본값(null, 0 등)으로 복원된다.

```java
private transient String password;   // 직렬화 제외 → 보안 데이터 보호
private transient Connection conn;   // 직렬화 불가능한 객체 제외
```

#### serialVersionUID

클래스의 **버전 식별자**. 직렬화된 데이터와 역직렬화할 클래스의 버전이 다르면 `InvalidClassException`이 발생한다.

```java
// 명시적으로 선언하지 않으면 JVM이 클래스 구조 기반으로 자동 생성
// → 필드 하나만 추가해도 UID가 바뀌어 기존 직렬화 데이터와 호환 불가
private static final long serialVersionUID = 1L;  // 명시적 선언 권장
```

### Java 기본 직렬화의 문제점

| 문제 | 설명 |
|------|------|
| **버전 호환성** | 클래스 구조 변경 시 역직렬화 실패 (serialVersionUID) |
| **보안** | 역직렬화 시 임의 코드 실행 가능 (역직렬화 공격) |
| **성능** | 리플렉션 기반으로 느림 |
| **크기** | 클래스 메타 정보까지 포함하여 데이터 크기가 큼 |
| **언어 종속** | Java 간에만 통신 가능 |

### 실무에서의 직렬화 — JSON, Protocol Buffers

실무에서는 Java 기본 직렬화 대신 **JSON**이나 **Protocol Buffers** 등을 사용한다.

```java
// Jackson — JSON 직렬화
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(user);     // 객체 → JSON
User restored = mapper.readValue(json, User.class); // JSON → 객체
```

| | Java Serializable | JSON (Jackson) | Protocol Buffers |
|---|---|---|---|
| 포맷 | 바이너리 (Java 전용) | **텍스트** (범용) | **바이너리** (범용) |
| 가독성 | 없음 | **높음** | 없음 |
| 크기 | 큼 | 중간 | **작음** |
| 속도 | 느림 | 중간 | **빠름** |
| 언어 호환 | Java만 | **모든 언어** | **모든 언어** |
| 사용처 | 레거시 | **REST API, 웹** | **gRPC, 내부 통신** |
