# 큐 (Queue)
데이터를 저장하는 **선형 자료구조**로, 선입선출(First In First Out) 방식을 따른다.

핵심 연산
- enqueue() : 데이터를 제일 마지막에 추가
- dequeue() : 가장 먼저 들어온 데이터를 제거

## ArrayDeque vs LinkedList

| 항목          | ArrayDeque           | LinkedList         |
|-------------|----------------------|--------------------|
| 구조          | 동적 배열 기반             | 이중 연결 리스트          |
| 삽입/삭제 (양 끝) | O(1) (암시적 포인터 이동)    | O(1) (명시적 포인터 수정 ) |
| 메모리 효율      | 캐시 친화적, 데이터가 메모리상 인접 | 포인터 객체로 분산됨        |
| GC 부담       | 낮음 (순차 배열)           | 높음 (객체 다수 생성)      |
| 성능 최적화 여부   | JDK 내부에서 최적화 많이 됨    | 상대적으로 덜 됨          |

## Queue 인터페이스
| 기능 | 예외 발생     | 특수 값 반환  | 설명                        |
|----|-----------|----------|---------------------------|
| 삽입 | add(e)    | offer(e) | offer는 실패시 `false` 반환     |
| 삭제 | remove()  | poll()   | poll는 큐가 비어있으면 `null`을 반환 |
| 확인 | element() | peek()   | peek은 큐가 비어있으면 `null`을 반환 |
