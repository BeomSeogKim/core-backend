# 스택 (Stack)
데이터를 순서대로 쌓아 올리는 선형 자료구조

가장 나중에 들어온 데이터가 가장 먼저 나가는 후입선출 (LIFO, Last In First Out) 원칙을 따른다.

## 핵심 개념

핵심 연산
 - push : 데이터가 스택의 맨 위에 쌓이는 과정
 - pop : 스택의 맨 위에서 데이터가 빠져 나오는 과정

## Stack 두 개로 Queue 구현

Stack 두 개를 활용하면 FIFO 순서를 만들 수 있다.

- **enqueue**: Stack1에 push
- **dequeue**: Stack2가 비어있으면 Stack1에서 전부 pop → Stack2로 push (순서 뒤집힘). Stack2에서 pop

## ArrayDeque vs Stack

ArrayDeque (권장)
- 동적 배열을 양쪽 끝에서 데이터를 처리할 수 있는 `Deque`의 구현체
- 스택의 모든 기능을 효율적으로 지원
Stack (비권장)
- Java의 초기 버전부터 존재했던 클래스
- 오래된 `Vector` 클래스를 상속받아 불필요한 동기화 처리로 인한 성능 저하가 있음
