# 해시 테이블 (Hash Table)
`Key - Value` 쌍으로 데이터를 저장하는 자료구조

평균적으로 O(1) 이라는 매우 빠른 시간 복잡도로 데이터를 삽입, 삭제 및 조회를 할 수 있다.

## 동작원리
> 해시 함수 (Hash Function)를 사용해 어떤 Key라도 배열의 특정 인덱스로 변환

핵심 요소
- 해시 함수
    - 다양한 종류와 길이의 Key(문자열, 숫자 등등..)를 입력받아, 배열의 인덱스로 사용할 수 있는 고정된 범위의 정수로 변환하는 함수
- 버킷
    - 해시 테이블에서 데이터를 저장하는 각각의 공간을 일컫는다.

***Key만 있으면 해시 함수가 데이터의 위치를 바로 알려주기 때문에 평균 O(1)로 빠르게 찾을 수 있다.***

## Collision
서로 다른 key값이 동일한 해시값을 가질 때 발생하는 상황.

이를 해결하기 위한 방법으로 `Open Addressing`, `Seperate Chaining` 방식이 존재한다.
