# 해시 테이블 (Hash Table)
`Key - Value` 쌍으로 데이터를 저장하는 자료구조

평균적으로 O(1) 이라는 매우 빠른 시간 복잡도로 데이터를 삽입, 삭제 및 조회를 할 수 있다.

## 동작원리
> 해시 함수 (Hash Function)를 사용해 어떤 Key라도 배열의 특정 인덱스로 변환

핵심 요소
- 해시 함수
    - 다양한 종류와 길이의 Key(문자열, 숫자 등등..)를 입력받아, 배열의 인덱스로 사용할 수 있는 고정된 범위의 정수로 변환하는 함수
- 버킷
    - 해시 테이블에서 데이터를 저장하는 각각의 공간을 일컫는다.

***Key만 있으면 해시 함수가 데이터의 위치를 바로 알려주기 때문에 평균 O(1)로 빠르게 찾을 수 있다.***

## Collision
서로 다른 key값이 동일한 해시값을 가질 때 발생하는 상황.

이를 해결하기 위한 방법으로 **Separate Chaining**과 **Open Addressing** 방식이 존재한다.

### Separate Chaining (분리 연결법)

같은 해시값을 가진 데이터를 **Linked List**로 연결하여 저장하는 방식.

### Open Addressing (개방 주소법)

충돌 시 **빈 슬롯을 탐사(Probing)**하여 데이터를 저장하는 방식.

탐사 방법:
- **Linear Probing**: 충돌 시 바로 다음 칸 (+1, +2, +3...) → **Primary Clustering** 문제 발생 가능
- **Quadratic Probing**: 제곱 간격 (+1, +4, +9...)
- **Double Hashing**: 두 번째 해시 함수로 간격 결정

### Separate Chaining vs Open Addressing

| | Separate Chaining | Open Addressing |
|--|-------------------|-----------------|
| **장점** | 삭제 용이, 적재율 1 이상 가능 | 메모리 효율적 (포인터 불필요) |
| **단점** | 포인터 메모리 오버헤드 | Clustering, 삭제 복잡, 적재율 높으면 성능 급락 |
| **사용처** | Java HashMap | Python dict, Go map |

### Java HashMap의 최적화

Java의 `HashMap`은 **Separate Chaining** 방식을 채택한다. 충돌이 심해지면 내부적으로 자료구조를 변환한다:

- 충돌 **8개 이상**: Linked List → **Red-Black Tree**로 변환 (O(n) → O(log n))
- 충돌 **6개 이하**: 다시 Linked List로 복원
- 8/6 간격을 둔 이유: 경계값에서의 반복 변환(thrashing) 방지
