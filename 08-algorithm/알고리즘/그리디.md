# 그리디 (Greedy)

## 핵심 개념

> [!note] 핵심 요약
> **그리디 알고리즘**은 각 단계에서 **현재 최적의 선택**을 반복하여 전체 최적해를 구하는 방법이다. DP와 달리 이전 선택을 되돌리지 않으며, 문제가 **탐욕적 선택 속성**과 **최적 부분 구조**를 만족할 때 사용할 수 있다.

### 그리디가 적용 가능한 조건

1. **Greedy Choice Property** — 현재 최선의 선택이 전체 최적해에 포함된다
2. **Optimal Substructure** — 부분 문제의 최적해가 전체 최적해를 구성한다

## 동작 원리

### 일반적인 그리디 패턴

```
1. 문제를 정렬하거나 우선순위를 정한다
2. 가장 좋은 선택을 한다
3. 선택이 조건을 만족하면 포함, 아니면 버린다
4. 반복
```

### 그리디 vs DP

| 항목 | 그리디 | DP |
|------|--------|-----|
| 선택 방식 | 현재 최적만 고려 | 모든 경우 고려 |
| 되돌림 | 없음 | 부분 문제 재활용 |
| 시간복잡도 | 보통 O(N) or O(N log N) | 보통 O(N²) 이상 |
| 적용 조건 | 탐욕적 선택 속성 필요 | 최적 부분 구조만 있으면 됨 |

## 코드 예시

### Frog Jump — 최소 점프 횟수

X에서 Y까지 D씩 뛸 때 최소 점프 수:

```java
public int frogJump(int X, int Y, int D) {
    return (Y - X) % D == 0 ? (Y - X) / D : (Y - X) / D + 1;
}
```

> [!tip] 면접 팁
> **올림 나눗셈** 패턴: `(a + b - 1) / b`로도 쓸 수 있다. `(Y - X + D - 1) / D`와 동일한 결과.

### 배열 Rotate — 모듈러 연산 활용

```java
public int[] rotate(int[] A, int K) {
    if (A.length == 0) return A;

    int[] result = new int[A.length];
    for (int i = 0; i < A.length; i++) {
        int index = (i + K) % A.length;
        result[index] = A[i];
    }
    return result;
}
```

> [!tip] 면접 팁
> **모듈러 연산** `%`은 순환 구조(원형 배열, 시계 등)에서 인덱스를 다룰 때 핵심이다. K가 배열 길이보다 커도 `% length`로 자동 처리된다.

### 동전 교환 (대표적 그리디)

```java
// 가장 큰 동전부터 사용
public int minCoins(int[] coins, int amount) {
    Arrays.sort(coins);
    int count = 0;
    for (int i = coins.length - 1; i >= 0; i--) {
        count += amount / coins[i];
        amount %= coins[i];
    }
    return amount == 0 ? count : -1;
}
```

> [!warning] 주의
> 동전 교환은 동전 종류에 따라 그리디가 최적해를 보장하지 않을 수 있다. `[1, 3, 4]`에서 6을 만들 때 그리디는 `4+1+1=3개`이지만 최적은 `3+3=2개`. 이 경우 DP가 필요하다.
