# 정렬 (Sorting)

## 핵심 개념

> [!note] 핵심 요약
> 정렬은 데이터를 특정 기준에 따라 순서대로 배치하는 알고리즘이다. 대표적인 O(n log n) 정렬로 **Merge Sort**, **Quick Sort**, **Heap Sort**가 있으며, 각각 안정성, 평균 성능, 공간 효율에서 서로 다른 트레이드오프를 가진다.

### 시간복잡도 비교

| 정렬 | Best | Average | Worst | Space |
|------|------|---------|-------|-------|
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) |
| Quick Sort | O(n log n) | O(n log n) | O(n^2) | O(log n) |
| Heap Sort | O(n log n) | O(n log n) | O(n log n) | O(1) |

## 동작 원리

### Merge Sort (합병 정렬)

**쪼개고 -> 합치면서 정렬**

1. 배열을 절반씩 [[재귀]]적으로 쪼갠다 (원소 1개가 될 때까지)
2. 두 포인터로 비교하면서 정렬된 상태로 합친다

```
[5, 3, 1, 4]
      | 절반으로 쪼개기
 [5, 3]   [1, 4]
    |          |
 [5] [3]   [1] [4]
    |          |
  [3, 5]   [1, 4]
      | 합치면서 정렬
   [1, 3, 4, 5]
```

> [!tip] 시간복잡도 유도
> - 쪼개는 단계 수 = **log n** (매번 절반으로 나누므로)
> - 각 단계에서 비교 횟수 = **n** (모든 원소를 한 번씩 비교)
> - **O(n log n)**, Worst case도 항상 동일

> [!warning] 공간복잡도 O(n)
> 합칠 때 임시 배열(temp)이 필요하기 때문에 추가 메모리가 필요하다.

### Quick Sort (퀵 정렬)

**pivot 기준으로 쪼개면서 정렬**

1. pivot(기준값) 하나를 선택
2. pivot보다 작은 것 -> 왼쪽, 큰 것 -> 오른쪽
3. pivot은 제자리 확정 (더 이상 이동 없음)
4. 양쪽에 [[재귀]]적으로 반복

```
[5, 3, 1, 4, 2]  pivot=3
      |
[1, 2]  [3]  [4, 5]
  |               |
pivot=1         pivot=4
[] [1] [2]    [] [4] [5]
      |
[1, 2, 3, 4, 5]
```

> [!tip] 시간복잡도 유도
> - **Average O(n log n)**: pivot이 중간값이면 Merge Sort처럼 log n 단계
> - **Worst O(n^2)**: pivot이 항상 최솟값/최댓값이면 한 번에 원소 1개만 확정
>   - 예) 이미 정렬된 배열 `[1,2,3,4,5]`에서 첫 원소를 pivot으로 선택할 때

> [!note] 공간복잡도 O(log n)
> [[재귀]] 호출 스택 깊이만큼 사용 (in-place 정렬이라 추가 배열 불필요)

### Heap Sort (힙 정렬)

**Max-Heap의 루트 = 최댓값** 을 이용

- **Max-Heap 규칙**: 부모 노드가 항상 자식 노드보다 크다
- 루트(최댓값)를 꺼내서 배열 맨 뒤에 놓기 -> 반복

```
[5, 3, 1, 4, 2]
Max-Heap 구성 -> 루트 = 5
5 꺼내서 맨 뒤  -> [?, ?, ?, ?, 5]
재구성 -> 루트 = 4
4 꺼내서 맨 뒤  -> [?, ?, ?, 4, 5]
...
[1, 2, 3, 4, 5]
```

> [!tip] 시간복잡도 유도
> - 루트 꺼내는 횟수 = **n번**
> - 매번 Heap 재구성 비용 = 트리 높이 = **log n**
> - **O(n log n)**, Worst case도 항상 동일

> [!note] 공간복잡도 O(1)
> 기존 배열 내에서 in-place로 처리, 추가 공간 불필요

## 코드 예시

### Merge Sort 구현

```java
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;  // base case: 원소 1개

    int mid = (left + right) / 2;
    mergeSort(arr, left, mid);      // 왼쪽 절반
    mergeSort(arr, mid + 1, right); // 오른쪽 절반
    merge(arr, left, mid, right);
}

void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;

    // 두 포인터로 비교하면서 작은 것부터 선택
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }

    // 남은 원소 처리
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    // temp -> arr 복사
    for (int l = 0; l < temp.length; l++) {
        arr[left + l] = temp[l];
    }
}
```
