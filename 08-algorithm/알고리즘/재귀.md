# 재귀 (Recursion)
어떠한 문제를 자기와 동일한 구조를 가진 더 작은 하위 문제로 나누어 해결하는 방식

재귀함수는 두가지 요소로 구성된다.

1. Base Case (종료 조건)
    - 더 이상 자기를 호출하지 않고 종료되는 조건.
2. Recursive Case (재귀 호출)
    - 문제를 더 작은 문제로 나누어 자기 자신을 호출하는 부분

## Basecase
> 재귀함수가 더 이상 자신을 호출하지 않고 종료되는 조건으로, 재귀 호출을 멈추게 만드는 명확한 종료 시점

### Basecase 지정 3가지 원칙
#### 1. 수렴 조건 (Convergence Condition)
- 재귀 호출 시마다 입력값이 점점 작아지거나 단순해짐
- 결국에는 base case에 도달

#### 2. 간단한 답 (Trivial Case)
- 이미 정답이 명확한 경우, 바로 반환

#### 3. 조건부 호출 (Conditional Recursion)
- 재귀 호출을 특정 조건에서만 실행
- 기저 조건이 명시적이지 않아도 자연스럽게 재귀를 제한

## 데이터 전달 방법

### 매개변수
함수를 매번 호출 할 때 마다 필요한 정보를 매개변수로 전달하여 호출마다 달라지는 정보를 저장하는 방식

### 반환값
재귀 함수에서의 반환값은 하위 호출의 결과를 받아 누적하거나 가공하는 과정의 핵심값

- return이 없으면 계산된 결과가 호출한 상위 함수에 전달되지 않음
- 재귀의 본질은 하위에서 계산된 값을 상위가 전달받아 이어서 처리하는 것이기 때문에, return 값이 필요하다.
