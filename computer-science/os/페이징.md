# 페이징 (Paging)

## 핵심 개념

**페이징(Paging)**은 가상 메모리를 고정 크기 단위(**Page**)로, 물리 메모리를 같은 크기 단위(**Frame**)로 나누어 관리하는 메모리 관리 기법이다. 연속된 물리 공간 없이도 페이지 단위로 흩어진 프레임에 매핑할 수 있어 **External Fragmentation(외부 단편화)**을 근본적으로 해결한다.

## 동작 원리

### 왜 Paging이 필요한가 — External Fragmentation

가변 크기로 메모리를 할당하면, 프로세스가 종료될 때 빈 공간이 조각나서 **전체 여유 공간은 충분한데 연속 공간이 부족해 할당 불가능**한 상황이 발생한다:

```
[  A (5MB)  ][  빈 (3MB)  ][  C (7MB)  ][  빈 (2MB)  ]

전체 빈 공간: 5MB
D (4MB) 올리려면? → 연속 4MB 없음 → 할당 실패!

이것이 External Fragmentation (외부 단편화)
```

Paging은 이 문제를 **고정 크기 분할 + 비연속 할당**으로 해결한다.

### Page와 Frame

| 구분 | 대상 | 설명 |
|------|------|------|
| **Page** | 가상 메모리 | 가상 주소 공간을 고정 크기(보통 4KB)로 자른 단위 |
| **Frame** | 물리 메모리 | 물리 RAM을 같은 고정 크기(4KB)로 자른 단위 |

프로세스에 Page 3개가 필요하면, 연속되지 않은 Frame 3개를 아무 위치에서 골라 매핑하면 된다:

```
가상 메모리 (프로세스 A)              물리 메모리 (RAM)

┌────────────┐                   ┌────────────┐
│  Page 0    │──────────────────→│  Frame 3   │
├────────────┤                   ├────────────┤
│  Page 1    │──────┐            │  Frame 4   │ (다른 프로세스)
├────────────┤      │            ├────────────┤
│  Page 2    │──┐   └───────────→│  Frame 7   │
└────────────┘  │                ├────────────┤
                │                │  Frame 9   │ (다른 프로세스)
                │                ├────────────┤
                └───────────────→│  Frame 12  │
                                 └────────────┘

→ 연속된 Frame이 아니어도 상관없음 = 외부 단편화 없음
```

### Page Table

**Page Table**은 Page → Frame 매핑 정보를 저장하는 테이블이다. **프로세스마다 하나씩** 존재하며 메모리(RAM)에 저장된다.

```
프로세스 A의 Page Table:

가상 Page  →  물리 Frame  |  유효 비트 (Valid Bit)
──────────────────────────────────────────────
 Page 0    →  Frame 3     |    1  (RAM에 있음)
 Page 1    →  Frame 7     |    1
 Page 2    →  Frame 12    |    1
 Page 3    →  Disk 주소    |    0  (RAM에 없음 → Page Fault)
```

- **유효 비트 = 1**: 해당 Page가 RAM에 있음 → 바로 물리 주소 변환
- **유효 비트 = 0**: 해당 Page가 디스크에 있음 → **Page Fault** 발생 → OS가 디스크에서 RAM으로 로드

### 주소 변환 구조: MMU와 TLB

메모리 접근 시 가상 주소 → 물리 주소 변환이 필요하다. 이를 담당하는 하드웨어 구조:

```
┌─────────────────────────────────────────────────────┐
│                      MMU                            │
│              (Memory Management Unit)               │
│                                                     │
│   ┌─────────────────────────────────┐               │
│   │     TLB (Translation Lookaside  │               │
│   │          Buffer)                │               │
│   │                                 │               │
│   │  최근 Page→Frame 매핑 캐시       │               │
│   │  Hit → 바로 물리 주소 반환 (~1ns)│               │
│   └─────────────────────────────────┘               │
│                                                     │
│   Miss → 메모리의 Page Table 조회 (~100ns)          │
└─────────────────────────────────────────────────────┘
```

| 구성 요소 | 위치 | 역할 |
|-----------|------|------|
| **Page Table** | 메모리 (RAM) | Page → Frame 매핑 정보 저장 |
| **MMU** | CPU 칩 내 하드웨어 | Page Table을 참조하여 주소 변환 수행 |
| **TLB** | MMU 내부 캐시 | 최근 변환 결과를 캐싱하여 속도 향상 |

### 주소 변환 흐름

```
CPU: 가상 주소 접근
      │
      ▼
  ┌──────────┐
  │ TLB 조회  │
  └────┬─────┘
       │
  ┌────┴────┐
  │         │
 Hit       Miss
  │         │
  ▼         ▼
물리 주소  Page Table 조회 (메모리 접근)
반환 (~1ns) │
            ├── 유효 비트 = 1 → 물리 주소 반환 + TLB에 캐싱
            │
            └── 유효 비트 = 0 → Page Fault
                                │
                                ▼
                         OS가 디스크 → RAM 로드
                         Page Table 갱신
                         TLB 갱신
                         재시도
```

> [!tip] TLB Hit Rate
> TLB hit rate는 보통 **95% 이상**이다. 대부분의 메모리 접근은 TLB에서 해결되므로 Page Table로 인한 성능 저하는 실제로 크지 않다.

### Internal Fragmentation (내부 단편화)

Paging은 외부 단편화를 해결하지만, **Internal Fragmentation(내부 단편화)**는 발생할 수 있다.

Page 크기가 4KB인데 프로세스가 4.1KB를 요구하면:

```
┌──────────────────────┐
│ Page 0: 4KB (꽉 참)   │
├──────────────────────┤
│ Page 1: 0.1KB 사용    │
│         3.9KB 낭비 ◀  │  ← Internal Fragmentation
└──────────────────────┘

→ 마지막 Page 내부에 사용하지 않는 공간이 생김
```

하지만 낭비되는 양이 최대 (Page 크기 - 1) = **최대 4KB 미만**으로 한정되므로, 외부 단편화에 비해 문제가 훨씬 작다.

| 단편화 유형 | 원인 | Paging에서 |
|------------|------|-----------|
| **External (외부)** | 가변 크기 할당 → 빈 공간 조각남 | 발생 안 함 |
| **Internal (내부)** | 고정 크기 할당 → 마지막 Page 내부 낭비 | 발생 (최대 4KB 미만) |
