# 데드락 (Deadlock)

## 핵심 개념

**데드락(Deadlock, 교착상태)**은 두 개 이상의 스레드가 각자 자원을 보유한 채 상대방의 자원을 기다리면서 무한히 대기하는 상태다. 4가지 조건이 **동시에** 성립해야 발생한다.

### 데드락 발생 4가지 조건 (Coffman Conditions)

네 가지가 **모두 동시에** 만족해야 데드락이 발생한다. 하나라도 깨면 데드락은 발생하지 않는다.

### 데드락 해결 전략

#### 1. 예방 (Prevention) — 조건 자체를 깨기

| 깨는 조건 | 방법 | 현실성 |
|-----------|------|--------|
| 순환 대기 | 자원에 번호를 부여하고 **오름차순으로만 요청** | 가장 실용적 |
| 점유와 대기 | 필요한 자원을 **한꺼번에 모두 요청** | 자원 낭비 가능 |
| 비선점 | 자원 못 얻으면 **보유한 것도 반납** 후 재시도 | 구현 복잡 |
| 상호 배제 | 자원 공유 허용 | 대부분 불가능 |

#### 2. 회피 (Avoidance) — 안전한 경우에만 할당

**은행원 알고리즘 (Banker's Algorithm)**: 자원 할당 전에 "이 할당이 안전 상태(Safe State)를 유지하는가?"를 검사. 안전하지 않으면 할당을 보류한다.

#### 3. 탐지 & 회복 (Detection & Recovery)

데드락 발생을 허용하되, 주기적으로 탐지 후 해결:
- 프로세스 종료 (하나씩 또는 전체)
- 자원 선점 (특정 스레드의 자원을 강제 회수)

## 코드 예시

```java
// 데드락 발생 코드
Object lockA = new Object();
Object lockB = new Object();

// Thread 1
new Thread(() -> {
    synchronized (lockA) {
        // lockA 획득
        synchronized (lockB) {  // lockB 대기 → 데드락
            // ...
        }
    }
}).start();

// Thread 2
new Thread(() -> {
    synchronized (lockB) {
        // lockB 획득
        synchronized (lockA) {  // lockA 대기 → 데드락
            // ...
        }
    }
}).start();
```

```java
// 데드락 예방 — 순환 대기 조건 깨기 (자원 순서 통일)
// Thread 1
synchronized (lockA) {      // 항상 lockA 먼저
    synchronized (lockB) {
        // ...
    }
}

// Thread 2
synchronized (lockA) {      // 항상 lockA 먼저 (순서 통일)
    synchronized (lockB) {
        // ...
    }
}
```
